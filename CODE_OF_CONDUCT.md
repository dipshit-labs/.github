# Dipshit Labs Code of Conduct

## Our Pledge

We are here to ship code, learn things, and build cool stuff without the corporate bureaucracy. To make that happen, this department is a harassment-free zone for everyone—regardless of your background, experience level, or how many times you accidentally dropped the staging database. 

## The Rules of Engagement

### 1. Technical Standards

* If you break production, you fix production.
* "It worked on my machine" is not an acceptable pull request description. If it doesn't work in production, it doesn't work.
* Pushing directly to `main` is a crime against humanity. Use a branch.
* If your PR introduces spaghetti code, you are responsible for eating it (refactoring it) later. 
* When reviewing code, don't just say "this sucks." Explain *why* it sucks and provide a snippet on how to fix it.

### 2. Human Standards

* Trolling, personal attacks, insulting comments, and general toxicity will not be tolerated. We criticize the code, never the person. Don't be a dick.
* Do not leak private information, API keys, credentials, or personal conversations outside of this organization or the Discord server.
* Juniors should feel safe asking "stupid" questions. Seniors should answer them without being condescending.

## Enforcement

As the project organizers, I have the admin rights, and I will use them. If you repeatedly violate these rules, act toxic, or deliberately sabotage the workflow, you will be warned, muted on Discord, or kicked out of the GitHub Org entirely.
